Java Lab 09

Polymorphism.

Typed notes for polymorphism, method overloading, main method overloading, and constructor overloading.

What Is Polymorphism?

Polymorphism is a core concept in object-oriented programming that means many forms.

Difference Between Polymorphism and Overloading

  • Polymorphism is a concept or big idea in object-oriented programming.
  • Overloading is a type or way to achieve polymorphism.

Overloading

Overloading means multiple methods have the same name but different parameters.

Types of Overloading

  1. Method overloading
  2. Constructor overloading

Method Overloading

Method overloading means having multiple methods with the same name but different parameters.

The parameters can be different by type, number, or order.

Rules

  1. Method names should be the same.
  2. Number of parameters should be different.
  3. Data type of parameters should be different.
  4. Order of parameters should be different.

Data Type Note

If a method parameter is declared as double, the value should be passed as a decimal value.

sum(int x, double y);

sum(10, 20);    // wrong
sum(10, 20.5);  // correct

In overloading, Java strictly follows the parameter data type.

Method Overloading Example

class Addition {
    int a = 10;
    int b = 20;

    void sum() {
        System.out.println(a + b);
    }

    void sum(int x, int y) {
        System.out.println(x + y);
    }

    void sum(double x, double y) {
        System.out.println(x + y);
    }

    void sum(int x, double y) {
        System.out.println(x + y);
    }

    void sum(double y, int x) {
        System.out.println(x + y);
    }
}
  • sum() has no parameters.
  • sum(int x, int y) changes the number and type combination.
  • sum(double y, int x) uses a different parameter order.

Can We Overload the main Method?

Yes. Just like any other method, main() can be overloaded by changing the number or type of parameters.

Which main Method Is Executed by JVM?

The JVM only looks for this method:

public static void main(String[] args)
  • This is the starting point of the program.
  • Overloaded versions are not called automatically by JVM.

main Method Overloading Example

public class MainOverload {
    public static void main(String[] args) {
        System.out.println("Main method with String array");
        main(10);
        main("Jerish");
    }

    public static void main(int value) {
        System.out.println("Main method with int value");
    }

    public static void main(String name) {
        System.out.println("Main method with String value");
    }
}

Constructor Overloading

Constructor overloading means having multiple constructors in a class with different parameters.

Rules

  1. Constructor names should be the same as the class name.
  2. Number of parameters should be different.
  3. Data type of parameters should be different.
  4. Order of parameters should be different.

Constructor Overloading Example

class Box {
    double width;
    double height;
    double depth;

    Box() {
        width = height = depth = 0;
    }

    Box(double w, double h, double d) {
        width = w;
        height = h;
        depth = d;
    }

    Box(double len) {
        width = height = depth = len;
    }
}
  • Box() is the default constructor.
  • Box(double w, double h, double d) accepts three values.
  • Box(double len) accepts one value and assigns it to all dimensions.
