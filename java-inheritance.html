Inheritance

Acquiring all the properties, variables, and behavior methods from one class to another class is called inheritance.

Objective

  1. Re-usability
  2. Avoid duplication

How to Use Inheritance Concept?

When we use the extends keyword, all the methods and variables from that class can be used in the current class.

Single Inheritance Example

class Animal {
    void eat() {
        System.out.println("eating...");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println("barking...");
    }
}

Multilevel Inheritance Example

class Animal {
    void eat() {
        System.out.println("eating...");
    }
}

class Dog extends Animal {

        System.out.println("barking...");
    }
}

class Puppy extends Dog {

Multiple Inheritance Note

We cannot implement multiple inheritance using class concept. We can extend only one class at a time, not multiple classes.

Multiple inheritance can be achieved by interface concept.

Why Multiple Inheritance Using Classes Is Not Allowed

Java does not support multiple inheritance using classes because it can create ambiguity and the diamond problem.

Even if the parent classes do not have common methods, every class in Java extends the Object class by default. Both parent classes already contain inherited methods like toString(), equals(), and hashCode(). To avoid this complexity, Java allows a class to extend only one class, but it can implement multiple interfaces.

main Method Syntax

public static void main(String args[]) {

}
  • public: access modifier
  • static: directly called by JVM without object
  • void: no returned value
  • String args[]: String type array

Valid and Invalid main Method Examples

public static void main(String a[]) { }      // valid

public static void main(String []a) { }      // valid

void main(String args[]) public static { }   // invalid

public static void main(int a[]) { }         // not valid

static public void main(String args[]) { }   // valid

static void public main(String args[]) { }   // invalid
