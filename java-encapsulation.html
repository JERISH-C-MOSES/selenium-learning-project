Typed notes for variable types, this keyword, encapsulation, getters, setters, and static members.

Types of Variables

  1. Class variables or instance variables
  2. Local variables
  3. this keyword

Note: There is no global variable in Java.

Class and Instance Variables

Class variables or instance variables are declared inside a class but outside any method, constructor, or block.

public class ThisKeyword {
    int x, y; // class variables / instance variables

    void setData(int a, int b) { // a and b are local variables
        x = a;
        y = b;
    }
}

Local Variables

A local variable is declared inside a method, constructor, or block.

If local variables and class variables have the same name, Java gives priority to the local variables inside that method.

public class ThisKeyword {
    int x, y; // class variables / instance variables

    void setData(int x, int y) { // x and y are local variables
        x = x;
        y = y;
    }
}

To differentiate local variables and class variables, use the this keyword.

this Keyword

The this keyword represents the current class object.

It is used to differentiate local variables and class variables.

public class ThisKeyword {
    int x, y; // class variables / instance variables

    void setData(int x, int y) {
        this.x = x;
        this.y = y;
    }
}

What Is Encapsulation?

Encapsulation means keeping variables private and accessing them through public methods.

Normally we can assign a value to a variable using three methods:

  1. Using obj.variable
  2. Using methods
  3. Using constructors

Assigning value to a variable using object variables directly is not a good practice.

How to Achieve Encapsulation

  1. All variables should be private.
  2. For every variable there should be two methods: get and set.
  3. Variables can be operated only through methods.

Encapsulation Example

public class Account {
    private int accno;
    private String name;
    private double amount;

    void setAccno(int accno) {
        this.accno = accno;
    }

    int getAccno() {
        return this.accno;
    }
}
  • accno, name, and amount are private variables.
  • setAccno() assigns value to the private variable.
  • getAccno() returns the private variable value.

Static Keyword

A static variable in Java belongs to the class, not to individual objects. It is also called a class variable.

If a variable is declared with the static keyword, all objects share the same variable.

Why Use a Static Variable?

We use a static variable when the value should be common for all objects.

Important Point

A static variable can be accessed using the class name.

Student.college
System.out.println(Student.college);

If you change a static variable using one object, it changes for all objects because there is only one shared copy of that variable.

Static Method

A static method in Java belongs to the class, not to an object.

A static method can be called using the class name without creating an object.

A static method can call static methods and static variables directly. We do not need to create an object.

Static Keyword Rules

  1. Static methods can access static stuff directly without an object.
  2. Static methods can access non-static stuff through an object.
  3. Non-static methods can access everything directly, both static and non-static.

Static Keyword Example

public class StaticDemo {
    static int a = 10; // static variable
    int b = 20;        // non-static variable

    static void m1() { // static method
        System.out.println("this is m1 static method...");
    }

    void m2() { // non-static method
        System.out.println("this is m2 non-static method...");
    }

    public static void main(String[] args) {
        System.out.println(a);
        m1();

        StaticDemo demo = new StaticDemo();
        System.out.println(demo.b);
        demo.m2();
    }
}
