WebDriver Methods.

Theory-only notes for WebDriver methods.

Web Element

Definition

A web element is any individual component or object present on a web page.

Examples of web elements:

  • Text box
  • Button
  • Link
  • Checkbox
  • Radio button
  • Dropdown
  • Image
  • Table
  • Text label

These elements are created using HTML tags and are used to build and interact with a webpage.

Get Methods in Selenium WebDriver (Java)

Definition

Get methods are used to retrieve information from the browser or webpage using the WebDriver instance.

Get Methods in Selenium

Definition

Get methods are WebDriver methods used to retrieve information related to the browser or webpage, such as page title, URL, page source, and window IDs.

These methods work on the WebDriver instance and not on individual web elements.

1. get()

Purpose

Used to open a URL in the browser.

Syntax

driver.get("https://www.google.com");

2. getTitle()

Purpose

Returns the title of the current webpage.

Return Type

String

Syntax

String title = driver.getTitle();

3. getCurrentUrl()

Purpose

Returns the current webpage URL.

Return Type

String

Syntax

String url = driver.getCurrentUrl();

4. getPageSource()

Purpose

Returns the complete HTML source code of the webpage.

Return Type

String

Syntax

String source = driver.getPageSource();

5. getWindowHandle()

Purpose

Returns the unique ID of the current browser window.

Return Type

String

Syntax

String id = driver.getWindowHandle();

6. getWindowHandles()

Purpose

Returns IDs of all browser windows/tabs opened by Selenium.

Return Type

Set<String>

Syntax

Set<String> ids = driver.getWindowHandles();

Summary Table

MethodPurposeReturn Type
get()Opens URLvoid
getTitle()Returns page titleString
getCurrentUrl()Returns current URLString
getPageSource()Returns page sourceString
getWindowHandle()Returns current window IDString
getWindowHandles()Returns all window IDsSet<String>

Difference Between List and Set in Java

Why Set<String> is used in getWindowHandles()

Syntax

Set<String> ids = driver.getWindowHandles();

Reason

Each browser window/tab has a unique window ID.

Duplicate IDs should never exist.

So Selenium uses:

Set<String>

because:

  • Set stores only unique values.
  • Window IDs are unique.
  • No duplicate browser window IDs are possible.

getText() Definition

getText() is used to get the visible text present inside a web element.

It returns the text that is displayed on the webpage to the user.

If the webpage contains:

<h1 id="title">Welcome</h1>

Output:

Welcome

getAttribute() Definition

getAttribute() is used to get the value of an attribute from a web element.

Attributes are values inside HTML tags like:

  • id
  • class
  • value
  • name
  • href
  • placeholder

If the webpage contains:

<input id="email" type="text">

Output:

text

Difference Between getText() and getAttribute()

Feature getText() getAttribute()
Purpose Gets visible text Gets attribute value
Works On Text between tags HTML attributes
Returns Displayed text Attribute value
Example <button>Login</button> returns Login <input value="Admin"> returns Admin
Mainly Used For Validation of displayed text Fetching hidden or data values

Important Point

getText() does NOT work properly for input boxes because text inside input fields is stored in the value attribute.

For input fields use:

getAttribute("value")

NOT:

getText()

Conditional Methods in Selenium

Definition

Conditional methods are WebElement methods used to check the status or condition of a web element.

These methods return boolean values:

true / false

They are accessed through the WebElement interface.

Common Conditional Methods

MethodPurpose
isDisplayed()Checks whether the element is visible
isEnabled()Checks whether the element is enabled
isSelected()Checks whether the element is selected

Syntax

WebElement ele = driver.findElement(By.id("sample"));

ele.isDisplayed();
ele.isEnabled();
ele.isSelected();

Return Type

All conditional methods return:

boolean

Browser Methods in Selenium

Definition

Browser methods are WebDriver methods used to control browser windows and terminate browser sessions.

1. close()

Purpose

Closes the current browser window that Selenium is controlling.

Syntax

driver.close();

Important Point

  • Closes only the current window/tab.
  • WebDriver session may still remain active if other windows are open.

2. quit()

Purpose

Closes all browser windows opened by Selenium and ends the WebDriver session completely.

Syntax

driver.quit();

Important Point

  • Closes all tabs/windows.
  • Terminates WebDriver session completely.

Difference Between close() and quit()

Featureclose()quit()
Closes current windowYesYes
Closes all windowsNoYes
Ends WebDriver sessionNoYes
Mostly used forSingle window closingComplete browser termination

1. navigate().to()

Syntax

get()
driver.get("https://www.google.com");

Only String is allowed.

navigate().to()

String Format

driver.navigate().to("https://www.google.com");

URL Object Format

URL url = new URL("https://www.google.com");

driver.navigate().to(url);
  • get() -> accepts only String URL
  • navigate().to() -> accepts both String and URL object format

2. navigate().back()

Purpose

Moves browser to the previous webpage.

Syntax

driver.navigate().back();

Example

driver.navigate().back();

3. navigate().forward()

Purpose

Moves browser to the next webpage.

Syntax

driver.navigate().forward();

4. navigate().refresh()

Purpose

Refreshes/reloads the current webpage.

Syntax

driver.navigate().refresh();

Summary Table

MethodPurpose
navigate().to()Opens URL
navigate().back()Moves to previous page
navigate().forward()Moves to next page
navigate().refresh()Refreshes current page

Handling Browser Windows in Selenium

Definition

Window handling is used when Selenium needs to work with multiple browser windows or tabs.

By default, Selenium controls only the current window.

To perform actions on another window/tab, we must switch the driver control using:

driver.switchTo().window(windowID);

Important Methods

MethodReturn TypePurpose
getWindowHandle()StringReturns current window ID
getWindowHandles()Set<String>Returns all opened window IDs
switchTo().window()WebDriverSwitches control to selected window

Syntax

1. Get single/current window ID

String parentID = driver.getWindowHandle();

2. Get all window IDs

Set<String> windowIDs = driver.getWindowHandles();

3. Switch to another window

driver.switchTo().window(childID);

Example

//Click on link
driver.findElement(By.xpath("xpath")).click();

//Capture all window IDs
Set<String> windowIDs = driver.getWindowHandles();

//Convert Set into List
List<String> windowList = new ArrayList<>(windowIDs);

//Capture parent and child window IDs
String parentID = windowList.get(0);
String childID = windowList.get(1);

//Switch to child window
driver.switchTo().window(childID);

//Switch back to parent window
driver.switchTo().window(parentID);
