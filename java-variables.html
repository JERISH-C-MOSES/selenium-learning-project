Variables and Data Types

Variable

A variable is a named memory location used to store data.

Data type

A data type defines the type of data a variable can store.

int a = 10;
  • int is the data type.
  • a is the variable name.
  • 10 is the value stored in the variable.

Types of Data Types

Data types in Java are divided into two types.

  1. Primitive data types
  2. Non-primitive data types

Variable Naming Rules

  1. Variable names can contain letters, digits, underscores, and dollar signs.
  2. A variable name should usually begin with a letter.
  3. A variable name must not contain spaces.
  4. A variable name can also begin with $ or _.
  5. Variable names are case-sensitive, so myVar and myvar are different.
  6. Reserved Java keywords like int, class, and boolean cannot be used as variable names.

Primitive and Non-primitive Notes

  • Primitive data types store actual values directly.
  • Non-primitive data types store references to objects.
  • Primitive data types cannot store null because they store actual values directly.
  • String is non-primitive because it is a class in Java, and a String object stores references, not direct values.

Reference Variable

A reference variable stores the address or reference of an object instead of storing the actual object data.

Student s = new Student();
  • new Student() creates the object.
  • s is the reference variable.

An object is the actual entity created in memory. A reference is the variable that stores the address of that object.

char, Unicode, and ASCII

char is 2 bytes in Java because Java uses the Unicode character system.

Unicode characters require 16 bits, which is 2 bytes of memory.

char ch = 'A';

// Unicode value of A = 65

Unicode is a character encoding standard used in Java to represent characters from different languages and symbols.

ASCII is a character encoding system where each character is assigned a numeric value.

ASCII Unicode
Supports mainly English characters. Supports characters from all languages.
Uses 7 or 8 bits. Uses 16 bits or more.
Old standard. New standard.

Note: Java mainly uses Unicode for character representation. ASCII values are included inside Unicode.

Variable Declaration and Initialization

Variable declaration means creating a variable without assigning a value.

int a;

Variable initialization means creating a variable and assigning a value.

int a = 10;

Variables are mutable by default, so their values can be changed.

int a = 10;
a = 20;

System.out.println(a);
Output:
20

We can declare multiple variables in one line.

int a = 10, b = 20, c = 30;

Variable Scope and Lifetime

Variable scope defines the area in a program where a variable can be accessed or used.

  1. Local variable
  2. Instance variable

Variable lifetime refers to how long a variable exists in memory during program execution.

  • A local variable exists only while the method is executing.
  • An instance variable exists as long as the object exists.

Where Variables Are Stored in Memory

Local Variable and Instance Variable

A local variable is declared inside a method and can be used only within that method.

An instance variable is declared inside a class but outside methods and belongs to the object.

class Student {
    int age = 25; // instance variable

    void display() {
        int marks = 90; // local variable
    }
}

Variable Shadowing

Variable shadowing happens when a local variable or parameter has the same name as an instance variable, causing the local variable to hide the instance variable inside that scope.

class Student {
    int age = 25;

    void display() {
        int age = 30;
        System.out.println(age);
        System.out.println(this.age);
    }
}
Output:
30
25

To access the instance variable, use this.age.

Variable Hiding

Variable hiding occurs when a subclass declares a variable with the same name as a variable in the parent class.

class Parent {
    int a = 10;
}

class Child extends Parent {
    int a = 20;

    void display() {
        System.out.println(a);
        System.out.println(super.a);
    }
}
Output:
20
10
  • Child variable a hides parent variable a.
  • super.a accesses the parent variable.

Constant

A variable declared with final becomes a constant, which means the value cannot be changed.

final int a = 15;
  • final makes the value fixed.
  • After assigning 15, the value of a cannot be changed.
