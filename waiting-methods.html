Lab 02

Waiting Methods.

Theory-only notes for Selenium waiting concepts.

Wait Commands

Java wait methods - Thread.sleep();

Advantages of Thread.sleep()

  1. Easy to use and implement.

Disadvantages of Thread.sleep()

  1. If the wait time is insufficient, an exception may occur.
  2. It waits for the full specified time even if the element is ready earlier, which reduces script performance.
  3. Using it multiple times can make the script slow and inefficient.

Selenium Wait Methods

  1. Implicit wait
  2. Explicit wait / Fluent wait

Implicit Wait

Definition

Implicit wait tells Selenium to wait for a specified amount of time before throwing NoSuchElementException if the element is not found immediately.

Note: we just need to declare it once after the WebDriver and it will be applicable for each statement.

Syntax

driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));

How it works

  • Selenium waits up to 10 seconds to find the element.
  • If the element is found earlier, it continues immediately.
  • If not found within the time limit, it throws NoSuchElementException.

Advantages of Implicit Wait

  1. Needs to be written only once in the script.
  2. It does not wait for the full timeout if the element is found earlier.
  3. Applicable to all elements globally.
  4. Easy to use and implement.

Disadvantages of Implicit Wait

  1. If the specified wait time is insufficient, an exception may occur.
  2. May slow down execution when elements are not available.
  3. Not suitable for handling complex conditions.
  4. Mixing with explicit wait can cause unpredictable wait behavior.

Explicit Wait

Definition

Explicit wait is used to wait for a specific condition to occur before performing an action on an element.

It waits only for the targeted element and for the specified condition.

Syntax

WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));

wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("username")));

How it works

  • Selenium waits up to the given time.
  • It continuously checks for the condition.
  • Once the condition is satisfied, execution continues immediately.
  • If the condition is not met within the timeout, it throws TimeoutException.

Common ExpectedConditions

visibilityOfElementLocated()

elementToBeClickable()

presenceOfElementLocated()

alertIsPresent()

titleContains()

Method

  • presenceOfAllElementsLocatedBy()
  • visibilityOfAllElementsLocatedBy()
  • numberOfElementsToBe()

Important Point

In explicit wait, you can directly use ExpectedConditions with locator methods like:

By.id()
By.xpath()
By.name()

So, you do not need to use:

driver.findElement()

separately before waiting.

Advantages

  1. Waits only for the required element/condition.
  2. No need to use driver.findElement() separately in many cases.
  3. Improves script performance.
  4. More flexible and reliable.
  5. Does not wait for full timeout if condition is met early.

Disadvantages

  1. Need to write wait statements multiple times.
  2. Slightly more complex than implicit wait.
  3. Too many waits can increase code length.

Fluent Wait

Definition

Fluent Wait is a type of wait in Selenium that allows users to define:

  • Maximum wait time
  • Polling frequency
  • Exceptions to ignore while waiting

It repeatedly checks for the condition at regular intervals until the condition is met or the timeout occurs.

Wait<WebDriver> wait = new FluentWait<>(driver)
    .withTimeout(Duration.ofSeconds(20))
    .pollingEvery(Duration.ofSeconds(2))
    .ignoring(NoSuchElementException.class);

WebElement ele = wait.until(
    ExpectedConditions.elementToBeClickable(By.id("login"))
);

ele.click();

Explanation

  • Maximum wait time - 20 seconds
  • Polling interval - checks every 2 seconds
  • Ignores NoSuchElementException while waiting
  • Continues immediately once element becomes clickable
