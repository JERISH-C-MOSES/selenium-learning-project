How Java Code Runs

Java code cannot be understood directly by the processor. It must be converted step by step.

Java code -> Compiler -> Byte code -> JVM -> Machine code -> Processor
  • Compiler converts Java code into byte code.
  • Byte code is the code understood by the JVM.
  • JVM means Java Virtual Machine.
  • JVM converts byte code into machine-readable code.
  • Machine code is in binary format, like 0 and 1.

JDK, JRE, JVM, and JIT

JDK
  -> Java Compiler
  -> JRE
       -> JVM
       -> Library files
  • JDK means Java Development Kit.
  • JDK contains the Java compiler and JRE.
  • JRE means Java Runtime Environment.
  • JRE contains JVM and library files.
  • JIT means Just-In-Time compiler.
  • JIT is inside the JVM and helps save repeated compilation time.

First Java Program

Every Java program must be written inside a class.

class Hello {
    public static void main(String[] args) {
        System.out.println("Hello World");
    }
}
  • class Hello creates a class named Hello.
  • The file name must match the class name.
  • If the class name is Hello, the file name should be Hello.java.
  • main() is the starting point of the program.
  • JVM searches for the main() method first.
  • Each Java statement must end with a semicolon ;.
  • In System, the letter S must be capital.

Java Statements

A computer program is a list of instructions executed by a computer.

In programming language, these instructions are called statements.

  • Most Java programs contain many statements.
  • Some statements are understood as they are written.
  • Each statement usually ends with a semicolon ;.

Print Text

System.out.println() is used to output values or print text in Java.

System.out.println("Hello");
  • System.out.println() prints the text and moves the cursor to the next line.
  • System.out.print() prints the text and keeps the cursor on the same line.
  • Shortcuts like sop should be expanded and written clearly as System.out.println().

Double Quotes

Text must be wrapped inside double quotes. If double quotes are not given, an error will occur.

System.out.println("Hello");

This is valid because Hello is inside double quotes.

System.out.println(Hello);

This is invalid because Hello is not inside double quotes.

Print Numbers

Numbers can be printed without double quotes.

System.out.println(66);

Numbers can also be printed inside double quotes, but then Java treats them as text.

System.out.println("66");

Mathematical calculations can also be printed.

System.out.println(3 + 3);
System.out.println(3 * 3);

Java Comments

A comment is text inside code meant only for humans, not for the computer.

Java completely ignores comments when running the program.

Why We Use Comments

  • Explain what the code is doing.
  • Make code easy to understand later.
  • Leave notes for yourself or teammates.
  • Temporarily disable code without deleting it.

Single-Line Comment

A single-line comment starts with //. Everything after // on that line is ignored.

// This line is ignored
System.out.println("Hello");

Multi-Line Comment

A multi-line comment starts with /* and ends with */.

It is used for longer explanations.

/*
This program prints
Hello World
on the screen
*/
System.out.println("Hello World");

Documentation Comment

A documentation comment starts with /** and ends with */.

It can be read and documented by the Javadoc tool.

/**
 * This is the main method.
 * It is documented by Javadoc.
 */
public static void main(String[] args) {
    System.out.println("Hello");
}
