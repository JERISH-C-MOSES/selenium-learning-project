Object Oriented Programming.

Typed notes for OOP concepts, classes, objects, methods, and constructors in Java.

Object Oriented Programming

Java supports object oriented programming concepts.

  1. Class
  2. Object
  3. Polymorphism
  4. Encapsulation
  5. Inheritance
  6. Data Abstraction

Class

A class is a user-defined data type that contains variables and methods. It is used to create objects.

An object is an instance of a class that contains data and methods.

class ClassName {
    // variables
    // methods
}

Creating an Object

Use the class name, object name, and new keyword to create an object.

ClassName objectName = new ClassName();
Employee emp = new Employee();

When we create multiple objects of the same class, all objects are different and occupy different memory spaces.

Methods

A method is a block of code that performs a specific task.

  • If you do not want to repeat the same code again and again, write it once inside a method.
  • Then you can call the method whenever you want.
returnType methodName(parameters) {
    // code
}

Types of Methods

  1. No parameters and no return value.
  2. No parameters and return value.
  3. Takes parameters and no return value.
  4. Takes parameters and return value.

1. No Parameters and No Return Value

void methodName() {
    // code
}

Since the method is returning the same value, we can store it in a variable when we call it only if it has a return type. A void method does not return any value.

public class Greetings {
    void m1() {
        System.out.println("Hello");
    }

    public static void main(String[] args) {
        Greetings gr = new Greetings();
        gr.m1();
    }
}
Output:
Hello

2. No Parameters and Return Value

returnType methodName() {
    return value;
}

When the method returns a value, store the value in a variable when calling it.

public class Greetings {
    String m2() {
        return "Hello how are you";
    }

    public static void main(String[] args) {
        Greetings gr = new Greetings();
        String s = gr.m2();
        System.out.println(s);
    }
}
Output:
Hello how are you

3. Takes Parameters and No Return Value

void methodName(dataType param1, dataType param2) {
    // code
}

When a method takes parameters, we pass values while calling the method.

void m3(String name, int age) {
    System.out.println("Name is " + name + " Age is " + age);
}
Greetings gr = new Greetings();
gr.m3("Jerish", 29);
Output:
Name is Jerish Age is 29

4. Takes Parameters and Return Value

returnType methodName(dataType param1, dataType param2) {
    return value;
}
String m4(String name, int age) {
    return "Name is " + name + " Age is " + age;
}
Greetings gr = new Greetings();
String s1 = gr.m4("Jerish", 29);
System.out.println(s1);
Output:
Name is Jerish Age is 29

Since the method returns a string, store the returned data in a string variable and print it.

Student Method Example

Methods can be used to store data into variables and print the object data.

public class Student {
    int sid;
    String sname;
    char grad;

    void printStudentData() {
        System.out.println(sid + " " + sname + " " + grad);
    }

    public static void main(String[] args) {
        Student stu = new Student();
        stu.sid = 101;
        stu.sname = "John";
        stu.grad = 'A';
        stu.printStudentData();
    }
}

Constructor

A constructor in Java is a special method used to initialize objects. It is automatically called when an object is created.

Constructor Rules

  1. Constructor name should be the same as the class name.
  2. Constructor does not specify a return value or return type.
  3. Constructor can take parameters.
  4. Constructor is automatically invoked at the time of object creation.
  5. Constructor is used for initializing the values of variables.
class ClassName {
    ClassName() {
        // constructor
    }
}

Types of Constructors

  1. Default constructor
  2. Parameterized constructor

Default Constructor

Java creates a default constructor automatically when we do not create any constructor in the class.

  • It is not shown on the screen.
  • Object creation must use the default constructor.
  • It does not have any value in it unless we write it manually.
class Student {
    int x;
    int y;

    Student() {
        x = 100;
        y = 200;
    }
}

Parameterized Constructor

A parameterized constructor accepts arguments to initialize an object with specific values.

class ClassName {
    ClassName(dataType variable) {
        // code
    }
}
class Student {
    int id;
    String name;

    Student(int i, String n) {
        id = i;
        name = n;
    }
}

Constructor Example

In this example, both default constructor and parameterized constructor are used.

public class Construct {
    int a;
    int b;

    Construct() {
        a = 100;
        b = 200;
    }

    Construct(int x, int y) {
        a = x;
        b = y;
    }

    void print() {
        System.out.println(a + b);
    }

    public static void main(String[] args) {
        Construct s = new Construct();
        s.print();

        Construct s1 = new Construct(300, 500);
        s1.print();
    }
}
Output:
300
800
